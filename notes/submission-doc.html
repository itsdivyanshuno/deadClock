01 Problem Statement Selected

Challenge: The Last-Minute Life Saver

Students, professionals, and entrepreneurs face a universal productivity crisis. Deadlines — assignments, presentations, interviews, bill payments, and meetings — slip through the cracks not because people are careless, but because the tools they use are passive by design.

Existing productivity apps share a fundamental limitation:

  • They wait for you to panic. A blank input box and an empty list are not planning — they are potential stress made visible.
  • They demand manual effort. Sorting, tagging, and reorganizing tasks falls entirely on the user, defeating the purpose of a "helper."
  • They lack context. A generic reminder at 9 AM tells you a task exists; it does not tell you why it matters now or what to do about it.
  • They ignore the bigger picture. Long-term goals and daily to-dos live in separate silos, with no bridge between them.

The result is the same: a growing sense of overwhelm, missed commitments, and the nagging feeling that there should be a better way.

The question we asked: What if your productivity app didn't just remind you of your deadlines — but actively helped you complete them?

02 Solution Overview

deadClock is a conversation-first AI agent that transforms how people interact with their workload. Instead of forms, drop-downs, and empty checklists, users simply chat naturally — and the agent acts.

You: "I have a thesis Friday, a presentation tomorrow, and three reports due."
deadClock: Creates tasks → prioritizes by urgency → generates a time-blocked schedule → breaks down long-term goals → warns about at-risk deadlines → Done.

Built on Google Gemini function calling, deadClock has access to nine specialized AI tools that let it read, create, update, and analyze your tasks and goals in real time. Every state change flows through the agent, through an atomic SQLite transaction, and back to the UI — in a single round-trip.

What makes it different

  • Agentic, not automated. The AI reasons about your workload — it doesn't just fire reminders on a timer.
  • State-aware. It sees your full task history, understands priorities, and delivers context-aware recommendations every time.
  • Proactive by design. On every turn, a 5-pass analysis engine surfaces overdue items, focus gaps, and at-risk goals before you have to look for them.
  • Goal-connected. Long-term goals are broken into proportional weekly tasks automatically, so daily effort always maps to something bigger.

How the AI loop works

User message ──► POST /api/chat
                            │
                            ▼
                Gemini 2.5 Flash (full history)
                            │
                            ▼
                Function calls: add_task, prioritize,
                schedule, break_down_goal, …
                            │
                            ▼
                Atomic SQLite transaction
                (wipe + reinsert in one write)
                            │
                            ▼
                Gemini follow-up in plain language
                            │
                            ▼
                Client updates UI instantly

The client never touches the database directly. Every mutation is mediated by the AI, verified by the database, and surfaced to the user in natural language.

05 Google Technologies Utilized

Google Gemini is the core intelligence engine behind deadClock. The entire AI experience — from natural language understanding to autonomous task execution — is powered by Google's generative AI platform.

Gemini 2.5 Flash

The application uses gemini-2.5-flash accessed through the official @google/genai Node.js SDK (v2.10). This model provides:

  • Native function calling — structured tool invocations with typed parameters, enabling the agent to perform precise CRUD operations on tasks and goals.
  • Multi-turn conversation memory — the full chat history (capped at 100 messages) is sent to Gemini each turn, giving the agent persistent context across sessions.
  • Low latency, high throughput — Flash's optimized architecture keeps the chat experience snappy even with large context windows.
  • System prompt engineering — a detailed system prompt defines the agent's persona, tool schemas, prioritization rules, and behavioral conventions.

Nine AI Function-Calling Tools Gemini

Tool Purpose
add_task Create a task with title, deadline, priority, category, subtasks, and time estimate.
prioritize_tasks Stable-sort the task list: pending first → by urgency score → completed items sink to bottom.
complete_task Mark a task done by ID; triggers streak counter increment and achievement unlock checks.
add_goal Create a long-term goal with named milestones and a target completion date.
suggest_schedule Generate a 2-hour time-blocked daily plan starting at 9 AM, respecting per-task effort estimates.
get_reminders Surface tasks organized by urgency tier: overdue, due today, due tomorrow, and due this week.
suggest_proactive_actions 5-pass workload analysis: scan for overdue items, upcoming deadlines, focus gaps, at-risk goals, and large unplanned tasks.
break_down_goal Split a goal into proportional weekly tasks with automated deadline distribution across the goal's timeline.
reschedule_at_risk_tasks Identify tasks within the 24-hour risk window and propose extended deadlines with rationale.

Agent Architecture Google AI

  • Two-turn orchestration pattern — a user message triggers Gemini to produce function calls; those calls execute against the in-memory state; Gemini then produces a human-readable summary of what it did.
  • Full authority model. The agent has complete CRUD and planning authority over the user's data through the tool layer. No action is taken without the AI deciding it's the right one.
  • Proactive behavior. On every new turn (not just when asked), the agent runs the suggest_proactive_actions pass and surfaces actionable insights.
  • Zero LLM overhead for analytics. The deriveInsights() engine runs as pure O(n) TypeScript computation — streaks, trends, and category breakdowns are calculated locally, keeping Gemini calls focused on conversational reasoning.

+ Product Experience & Design

deadClock is designed to feel less like a productivity tool and more like a capable assistant sitting beside you. The experience is built around three principles:

  • Conversation over configuration. There are no onboarding wizards, no workspace setups, no empty states that demand input. You open the app and start talking.
  • Proactive, not reactive. The AI surfaces risks, focus gaps, and urgent items before the user has to hunt for them. It does the scanning so the user doesn't have to.
  • Immediate feedback. Every action — adding a task, completing a goal, marking something done — produces an instant visual response: updated cards, streak animations, achievement unlocks, heatmap cells.

Design details

  • Responsive layout. Collapsible sidebar, main canvas, and SPA-style view switching across Chat, Tasks, Goals, Dashboard, Analytics, Heatmap, Reflection, and Settings.
  • Motion and micro-interactions. Framer Motion 12 spring animations on view transitions, chat bubbles, and card hover states create a tactile, responsive feel.
  • Dark mode. Full token inversion across 50+ CSS design tokens, persisted to localStorage. Custom scrollbars and animated loading skeletons throughout.
  • Keyboard-first. Cmd/Ctrl+K command palette with vim-style navigation (G → C / T / G / D / A / H) for power users.
  • Accessible foundations. Built on Radix primitives (via shadcn/ui and Base UI), ensuring proper ARIA semantics, focus management, and keyboard operability.

+ Technical Implementation

Data layer

  • Atomic SQLite transactions. saveState() performs a wipe-and-reinsert inside a single transaction using ON CONFLICT upserts. The database is always in a consistent state — no partial writes, no dirty reads.
  • Six normalized tables. tasks, goals, chatHistory, daily_logs, user_stats, and achievements cover the full application state.
  • Canonical shapers. normalizeTask() and normalizeGoal() guarantee safe defaults before every INSERT, eliminating parameter-mismatch bugs.
  • Server-only runtime. better-sqlite3 is imported exclusively under export const runtime = "nodejs" — never exposed to client components.

AI layer

  • Single entry point. The chat() function in lib/agent.ts is the sole orchestrator — loads state, builds the Gemini request, dispatches function calls, and returns structured results.
  • Typed API surface. Every API route (chat, complete, analytics, reflection) returns well-typed JSON. The client is a thin consumer that never guesses at the shape of the data.
  • Strict TypeScript. Full strict-mode enforcement. any types are isolated to the CommonJS boundary of lib/db.js only.

Performance

  • Turbopack dev server for fast HMR during development and optimized production builds.
  • Server Components by default — minimal client-side JavaScript. Client components are opted in explicitly where interactivity requires it.
  • Zero-LLM analytics. The deriveInsights() engine computes streak trends, completion rates, and category breakdowns locally in O(n), with no API calls.
